# PRD — Трекер личного бюджета

**Product Requirements Document**
**Версия:** 1.0
**Дата:** Май 2026

---

## 1. Обзор продукта

**Название:** Мой Бюджет
**Тип:** Одностраничное веб-приложение (single-page HTML)
**Формат поставки:** Один автономный `.html`-файл без зависимостей от сервера

### Цель

Помочь пользователям отслеживать свои доходы и расходы, получать мгновенный анализ бюджета и вырабатывать здоровые финансовые привычки.

### Аудитория

Люди, которые хотят контролировать личные финансы: вести учёт обязательных и ежедневных расходов, видеть остаток и получать персональные рекомендации по оптимизации бюджета.

---

## 2. Цели и метрики

| Цель | Индикатор успеха |
|------|-----------------|
| Пользователь вводит доход за < 30 сек | Поле дохода — первый интерактивный элемент |
| Пользователь добавляет расход за < 15 сек | Форма добавления всегда видна, без модалок |
| Данные сохраняются между сессиями | localStorage — данные не теряются при закрытии |
| Анализ виден без прокрутки до него | Итоговая карточка — в верхней части страницы |

---

## 3. Функциональные блоки

### 3.1 Доход

- Одно числовое поле с префиксом валюты (₸)
- Изменение значения мгновенно пересчитывает все блоки
- Сохраняется автоматически

### 3.2 Обязательные расходы

Фиксированные ежемесячные статьи: аренда, кредит, подписки, коммунальные услуги и т.д.

**Поля записи:**
- Эмодзи-иконка (выпадающий список)
- Название (текст)
- Сумма (число, ₸)

**Действия:**
- Добавить запись
- Удалить запись (кнопка ✕)

### 3.3 Ежедневные расходы

Переменные траты: кафе, продукты, транспорт, развлечения и т.д.

**Поля записи:**
- Эмодзи-иконка
- Название
- Сумма (₸)

**Вкладки просмотра:**
- **Сегодня** — записи текущего дня
- **Вчера** — записи предыдущего дня
- **Неделя** — сумма за последние 7 дней

Новые записи всегда добавляются в текущий день (todayKey = ISO date).

### 3.4 Анализ расходов

#### Итоговая карточка (вверху страницы)
- Остаток = Доход − Все расходы
- Прогресс-бар: % потраченного от дохода

#### Тайлы аналитики
| Тайл | Формула |
|------|---------|
| Свободно | Доход − Все расходы (не < 0) |
| Расходы | Обязательные + Ежедневные (месяц) |
| В день | Ежедневные расходы / Кол-во дней с тратами |
| Сберегаю | Остаток / Доход × 100% |

#### Диаграмма (пончик / donut chart)
- Красный сегмент — обязательные расходы (% от дохода)
- Золотой сегмент — ежедневные расходы (% от дохода)
- Зелёный сегмент — остаток

#### Цель накоплений
- Пользователь вводит целевую сумму (₸)
- Прогресс-бар: Остаток / Цель × 100%
- Подпись: сколько ещё нужно накопить

#### Персональный совет (инсайт)
Динамический текст на основе данных:

| Условие | Совет |
|---------|-------|
| Расходы > Доход | Предупреждение о перерасходе |
| Сбережения ≥ 20% | Похвала, подтверждение |
| Сбережения 10–19% | Совет довести до 20% |
| Обязательные > 50% дохода | Совет по правилу 50/30/20 |
| Иначе | Совет откладывать 10% |

## 4. Хранение данных

Все данные хранятся в `localStorage` браузера под ключом `budget_v2`.

```json
{
  "income": 500000,
  "goal": 100000,
  "fixed": [
    { "id": 1748000000000, "emoji": "🏠", "name": "Аренда", "amount": 150000 }
  ],
  "daily": {
    "2026-05-27": [
      { "id": 1748000000001, "emoji": "☕", "name": "Кофе", "amount": 1200 }
    ]
  }
}
```

- Ключи ежедневных записей — ISO-дата (`YYYY-MM-DD`)
- `id` — `Date.now()` в момент добавления (уникальный идентификатор)

## 5. UX и дизайн

### Стиль
Дружелюбный, спокойный, корпоративный. Тёплая бежево-кремовая палитра с зелёным акцентом.

### Палитра

| Токен | Hex | Использование |
|-------|-----|---------------|
| `--bg` | `#F5F3EE` | Фон страницы |
| `--surface` | `#FDFCFA` | Карточки |
| `--accent` | `#2E7D5E` | Основной акцент (зелёный) |
| `--accent2` | `#C84B31` | Опасность, расходы |
| `--gold` | `#C9963A` | Выделение, ежедневные |
| `--ink` | `#1C1A17` | Основной текст |

### Типографика
- **DM Serif Display** — заголовки, числа
- **DM Sans** — интерфейсный текст, кнопки, подписи

### Адаптивность
- Максимальная ширина контента: `520px`, центрирована
- Боковые отступы на мобильном: `16px`
- Все элементы управления рассчитаны на тач-взаимодействие (min-height кнопок и полей: `42–48px`)

## 6. Технические требования

| Параметр | Значение |
|----------|----------|
| Формат | Один `.html`-файл |
| Зависимости | Google Fonts (CDN), нет JS-библиотек |
| Хранилище | `localStorage` |
| Совместимость | Chrome, Safari, Firefox (последние версии) |
| Мобильные браузеры | iOS Safari, Android Chrome |
| Сетевые запросы | Только загрузка шрифтов при первом открытии |

---

## 7. Ограничения текущей версии (v1.0)

- Нет авторизации и облачной синхронизации
- Данные привязаны к браузеру — не переносятся между устройствами
- Нет экспорта (PDF, CSV)
- Нет истории по месяцам (только текущий)
- Валюта фиксирована (₸)
